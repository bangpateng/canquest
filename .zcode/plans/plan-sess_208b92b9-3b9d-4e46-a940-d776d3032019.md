# Rencana Restruktur Wallet — Token List + Detail View

## Konsep
Ubah main wallet view dari single-page dashboard (balance + actions + transactions) → **token list view** (kartu token yang bisa diklik). History transaksi pindah ke **notification badge saja** (sudah ada).

**Klik kartu token** → detail view dengan balance + Send/Receive/Swap + (CC: activity history, non-CC: tanpa activity dulu).

## Kompromi Activity (rekomendasi production)
- **CC detail**: TETAP ada activity history (reuse `TransactionsView`). CC = token utama, user harus bisa cek "CC masuk/keluar darimana" tanpa bolak-balik badge.
- **Token non-CC detail** (saat Phase 2 swap live): balance + actions saja dulu. Backend belum support per-token tx filter → activity nanti.

## Yang TIDAK berubah
- Notification badge (`transaction-notifications.tsx`) — sudah ada, tetap global
- `/transactions` page — tetap ada (dedicated history page)
- Lock modal, wallet setup, wallet reconnect
- Backend endpoints — tidak ada perubahan (reuse existing)

## Risiko
- **Activity history di CC detail**: reuse `TransactionsView` — component ini punya prop `variant` (`embedded`/`page`) + `refreshKey`. Aman reuse.
- **Routing**: pattern `[tokenId]/page.tsx` sudah ada preseden (`transactions/[id]`). `(platform)` route group → `/wallet/cc` tanpa prefix.
- **Nav active state**: `pathname.startsWith('/wallet/')` (platform-shell.tsx:49) → `/wallet/cc` tetap highlight Wallet. Aman.
